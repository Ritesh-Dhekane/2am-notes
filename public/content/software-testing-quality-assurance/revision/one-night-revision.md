# STQA One-Night Revision (Study OS)

> [!TIP]
> This cheatsheet is designed for quick scanning 2 hours before the exam. Focus on the Bold keywords.

## 1. Testing Fundamentals
- **Verification**: "Are we building the product right?" (Process focus, reviews, walk-throughs).
- **Validation**: "Are we building the right product?" (Product focus, actual testing).
- **Quality Assurance (QA)**: Preventive, process-oriented.
- **Quality Control (QC)**: Corrective, product-oriented.

## 2. Testing Levels
1. **Unit Testing**: Testing individual components (Developers).
2. **Integration Testing**: Testing combined parts (Big Bang, Top-Down, Bottom-Up).
3. **System Testing**: Testing the whole system against requirements.
4. **Acceptance Testing**: Final check by users (Alpha, Beta).

## 3. Black Box vs White Box
- **Black Box**: No knowledge of internal code. Focus on Input/Output.
  - *Techniques*: BVA, ECP, Decision Tables.
- **White Box**: Knowledge of internal structure/logic.
  - *Techniques*: Path Coverage, Statement Coverage, Branch Coverage.

## 4. STLC (Software Testing Life Cycle)
1. Requirement Analysis
2. Test Planning
3. Test Case Development
4. Environment Setup
5. Test Execution
6. Test Cycle Closure

## 5. Selenium (Automation)
- **Selenium Suite**: IDE (Record/Playback), RC (Deprecated), WebDriver (Modern), Grid (Parallel).
- **Locators**: ID, Name, XPath, CSS Selector, Link Text.
- **Wait Types**: Implicit (Global), Explicit (Specific condition), Fluent (Polling).

## 6. Static vs Dynamic Testing
- **Static**: No code execution (Reviews, Inspections).
- **Dynamic**: Code execution (Functional, Non-functional).
